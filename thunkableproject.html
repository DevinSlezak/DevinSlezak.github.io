Thunkable Project

The purpose of the program is to quiz the user with trivia questions, providing a multiple choice section as well as two free response questions. The video demonstrates the program asking a multiple choice question, adding points to the score at the bottom of the screen since the user chose the correct answer. The input of the program is the user’s choice for one of the four buttons provided, and the output is depicted to be the new score that the user has.

The data contained in the variable represents the total score that the player had accumulated throughout the game. It keeps track of how many questions were answered correctly. The game runs off of the player’s score, being a main component of the program since it is supposed to quiz the player but also be considered a game, so using this list would help keep track of the amount of questions that were correct.

The procedure creates a variable that instead tracks the high scores of the user throughout the various times they have played the game. After the last question, the procedure and code check the players high score, if the score is higher past high scores kept in the app’s storage, it instead puts that new high score until it is beaten or the game is reset. The user’s score is examined after the final question of the game, and once the button for the question is clicked, which submits the answers, it checks the current score with past high scores on the app, and if the score is greater than the past high scores, it alerts the user, then showing the new current high score and then resetting the game. In the first call to the procedure, it would evaluate whether the current score of the user is greater than the high score, and if it is, the current score would be saved as the high score until it is beaten. The second call, which also evaluates the current score with the high score, instead resets the game by executing differing code that only occurs when the current score is lower than the high score. The conditions tested in the first call and second call are both evaluating whether the user’s score is greater than the current high score, and whether it is true or false, the code would execute a different segment that either marks down the new high score and resets or solely resets until a better score is made.
